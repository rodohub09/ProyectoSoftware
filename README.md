# ProyectoSoftware
## Introducción
En este proyecto se quiere hacer una aplicacion, haciendo usos de metodologias agiles, en este caso la metodologia Scrum, para encontrar o publicar objetos perdidos/encontrados, para estudiantes o autoridades del campus udec

## Visión del problema
Falta de una aplicación dedicado al encuentro de objetos perdidos dentro del campus

## Visión de la solucion
Crear una aplicación, en flutter, para publicar/crear reportes, con los datos que se proporcionen, que sean visibles en una lista pública en conjunto con un sistema de match. El desarrollo de esta aplicacion está prevista para 2 sprints

## Visión para el sprint 1
Una aplicación simple para poder:
- Crear/publicar reportes tanto de objetos perdidos como encontrados con los datos correspondiente
- Una lista para visualizar los reportes creados
- Un sistema de filtrado para los reportes
- Una interfaz para un futuro inicio de sesión (No funcional)
- Un bosquejo de un mapa del campus para en un futuro establecer puntos de recolección
  
## Visión para el sprint 2
Una aplicacion ya completa y probada con:
- Inicio de sesion ya funcional diferenciando autoridades, estudiantes del campus y público general
- Sistema "Match" que detecte posibles considencias entre objetos perdidos y encontrados
- Mapa, ya funcional, con los puntos de interés/recolección dentro del campus
- Mejoras del Sistema de reportes, para edición, visualizacion y eliminación de los mismos
- Persistencia en memoria para los reportes y perfiles
